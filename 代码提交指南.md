# FunLife 代码提交指南

## 📋 提交前准备

### 1. 清理构建文件
```bash
# Windows (PowerShell)
Remove-Item -Recurse -Force app/build
Remove-Item -Recurse -Force .gradle/build-cache

# 或使用 Gradle 清理
./gradlew clean
```

### 2. 检查 .gitignore
确保 `.gitignore` 文件已正确配置，避免提交构建文件和临时文件。

## 🚀 提交步骤

### 方式一：提交所有更改（推荐用于首次提交）

```bash
# 1. 添加所有源代码文件
git add app/src/

# 2. 添加配置文件
git add *.gradle.kts gradle.properties settings.gradle.kts
git add gradle/wrapper/

# 3. 添加文档
git add *.md

# 4. 添加 .gitignore
git add .gitignore

# 5. 提交
git commit -m "feat: 完整的 FunLife 应用实现

- 实现用户登录注册系统
- 实现转盘抽奖功能（含保底机制）
- 实现习惯追踪、目标管理、心情记录
- 实现纪念日管理和统计分析
- 实现商店系统和金币经济
- 完善数据持久化和导入导出功能"

# 6. 推送到远程仓库
git push origin main
```

### 方式二：分批提交（推荐用于增量更新）

```bash
# 提交核心功能
git add app/src/main/java/com/example/funlife/data/
git commit -m "feat: 添加数据层实现（数据库、DAO、模型）"

git add app/src/main/java/com/example/funlife/repository/
git commit -m "feat: 添加仓库层实现"

git add app/src/main/java/com/example/funlife/viewmodel/
git commit -m "feat: 添加 ViewModel 层实现"

git add app/src/main/java/com/example/funlife/ui/
git commit -m "feat: 添加 UI 层实现（所有界面和组件）"

git add app/src/main/java/com/example/funlife/utils/
git commit -m "feat: 添加工具类（用户会话、数据导入导出）"

git add app/src/main/java/com/example/funlife/navigation/
git add app/src/main/java/com/example/funlife/MainActivity.kt
git commit -m "feat: 添加导航和主活动"

git add *.md
git commit -m "docs: 添加项目文档"

git push origin main
```

## 📝 提交信息规范

使用语义化提交信息：

- `feat:` 新功能
- `fix:` 修复 bug
- `docs:` 文档更新
- `style:` 代码格式调整
- `refactor:` 代码重构
- `test:` 测试相关
- `chore:` 构建配置等

## ⚠️ 注意事项

### 不要提交的文件
- `app/build/` - 构建输出
- `.gradle/` - Gradle 缓存
- `.idea/` - IDE 配置（部分）
- `local.properties` - 本地配置
- `*.apk`, `*.aab` - 编译产物
- `*.class`, `*.dex` - 编译文件

### 必须提交的文件
- `app/src/` - 所有源代码
- `*.gradle.kts` - Gradle 配置
- `gradle/wrapper/` - Gradle Wrapper
- `*.md` - 项目文档
- `.gitignore` - Git 忽略配置

## 🔍 提交前检查

```bash
# 查看将要提交的文件（简洁版）
git diff --name-only

# 查看暂存区文件
git diff --cached --name-only

# 确认没有构建文件
git status | grep -E "(build/|\.class|\.dex|\.apk)"
```

## 📦 首次推送到新仓库

```bash
# 1. 初始化仓库（如果还没有）
git init

# 2. 添加远程仓库
git remote add origin <你的仓库地址>

# 3. 创建并切换到 main 分支
git branch -M main

# 4. 添加文件并提交
git add app/src/ *.gradle.kts gradle/ *.md .gitignore
git commit -m "feat: 初始化 FunLife 项目"

# 5. 推送到远程
git push -u origin main
```

## 🎯 快速提交命令（当前状态）

基于你当前的修改，可以使用：

```bash
# 添加所有修改的源代码
git add app/src/main/java/

# 添加文档
git add *.md

# 提交
git commit -m "feat: 完善用户系统和核心功能

- 集成用户登录注册功能
- 优化转盘抽奖机制
- 完善数据模型和数据库结构
- 更新项目文档"

# 推送
git push origin main
```

## 💡 提示

1. 首次提交建议使用"方式一"，一次性提交所有核心代码
2. 后续更新使用"方式二"，按功能模块分批提交
3. 提交前务必运行 `./gradlew clean` 清理构建文件
4. 使用有意义的提交信息，方便后续查看历史记录
