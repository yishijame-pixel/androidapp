# FunLife - 生活娱乐工具 Android 应用

## 📋 项目概述

FunLife 是一个基于 Jetpack Compose 和 Material Design 3 的 Android 应用，集成了多个实用的生活娱乐功能模块，包括用户系统、纪念日管理、幸运转盘、游戏计分、习惯追踪、目标管理、心情记录、商城系统等。

### 应用信息
- **应用名称**: FunLife
- **包名**: com.example.funlife
- **最低 SDK**: Android 8.0 (API 26)
- **目标 SDK**: Android 14 (API 34)
- **开发语言**: Kotlin
- **UI 框架**: Jetpack Compose
- **架构模式**: MVVM

## 🚀 快速开始

### 环境要求
- **Android Studio**: Hedgehog (2023.1.1) 或更高版本
- **JDK**: 8 或更高版本（推荐 JDK 17）
- **Android SDK**: API 26+
- **Gradle**: 8.1.4
- **Kotlin**: 1.9.20

### 克隆项目
```bash
git clone <repository-url>
cd FunLife
```

### 配置 SDK 路径
在项目根目录创建或编辑 `local.properties` 文件：
```properties
sdk.dir=<你的Android SDK路径>
```

例如：
```properties
sdk.dir=D\:\\Androidsdk
```

### 使用 Android Studio 运行

1. 打开 Android Studio
2. 选择 "Open an Existing Project"
3. 选择项目根目录
4. 等待 Gradle 同步完成
5. 点击 "Run" 按钮（绿色三角形）或按 `Shift+F10`
6. 选择模拟器或连接的设备
7. 应用将自动编译、安装并启动

### 使用命令行编译

#### Windows 系统

1. **编译 Debug 版本**
```bash
gradlew.bat clean assembleDebug
```

2. **安装到设备**
```bash
<SDK路径>\platform-tools\adb.exe install -r app\build\outputs\apk\debug\app-debug.apk
```

3. **启动应用**
```bash
<SDK路径>\platform-tools\adb.exe shell am start -n com.example.funlife/.MainActivity
```

#### 使用提供的批处理脚本

项目提供了便捷的批处理脚本：

```bash
# 编译应用
compile.bat

# 编译、安装并启动
build-and-install.bat
```

## 📱 使用指南

## 💾 数据库设计

### Room Database 版本: 12

### 数据表

1. **users** - 用户表
   - id, username, nickname, password, createdAt

2. **anniversaries** - 纪念日表
   - id, name, date, type, description, createdAt

3. **players** - 玩家表
   - id, name, score, createdAt

4. **game_history** - 游戏历史表
   - id, playerId, score, timestamp

5. **spin_wheel_templates** - 转盘模板表
   - id, name, options, createdAt

6. **spin_wheel_history** - 转盘历史表
   - id, mode, result, timestamp

7. **custom_spin_modes** - 自定义转盘模式表
   - id, name, costPerSpin, options

8. **guarantee_counters** - 保底计数器表
   - id, mode, count, lastResetTime

9. **habits** - 习惯表
   - id, name, description, frequency, createdAt

10. **goals** - 目标表
    - id, title, description, deadline, completed

11. **mood_entries** - 心情记录表
    - id, mood, note, timestamp

12. **shop_items** - 商品表
    - id, name, price, description, stock

13. **user_coins** - 用户金币表
    - id, userId, amount, lastUpdated

14. **user_preferences** - 用户偏好表
    - id, userId, theme, notifications

## 🐛 故障排除

### 常见问题

#### 1. Gradle 同步失败
**解决方案**:
- 检查网络连接
- 使用阿里云镜像（已在 settings.gradle.kts 中配置）
- 清理项目: `Build → Clean Project`
- 重新同步: `File → Sync Project with Gradle Files`

#### 2. 编译错误
**解决方案**:
- 确保 JDK 版本正确（推荐 JDK 17）
- 检查 SDK 路径配置
- 删除 `.gradle` 和 `build` 文件夹后重新编译

#### 3. 数据库版本冲突
**解决方案**:
- 卸载应用重新安装
- 或使用数据库迁移策略

#### 4. 转盘点击无反应
**已修复**: 
- 移除了复杂的自动触发机制
- 使用 `rememberCoroutineScope()` 替代错误的协程作用域
- 详见 `修复说明.md`

#### 5. 协程作用域错误
**已修复**:
- 使用 Compose 的 `rememberCoroutineScope()`
- 避免使用 `CoroutineScope(Dispatchers.Main)`
