## 环境准备

### 必需工具
- **JDK 17 或更高版本**
- **Android SDK**（已配置在 `local.properties` 文件中）
- **Android 模拟器** 或 **真实设备**（已启用 USB 调试）

### 验证环境配置

1. 检查 JDK 版本：
```bash
java -version
```
应显示 Java 17 或更高版本。

2. 检查 Android SDK 路径：
打开项目根目录下的 `local.properties` 文件，确认 SDK 路径正确：
```properties
sdk.dir=D\:\\Androidsdk
```

## 检查设备连接

### 启动模拟器（如果使用模拟器）

1. 列出可用的模拟器：
```bash
D:\Androidsdk\emulator\emulator.exe -list-avds
```

2. 启动指定的模拟器：
```bash
D:\Androidsdk\emulator\emulator.exe -avd <模拟器名称>
```

例如：
```bash
D:\Androidsdk\emulator\emulator.exe -avd Pixel_5_API_30
```

### 检查设备连接状态

使用 ADB（Android Debug Bridge）检查已连接的设备：

```bash
D:\Androidsdk\platform-tools\adb.exe devices
```

**预期输出：**
```
List of devices attached
emulator-5554   device
```

如果显示 `device`，说明设备已正确连接。

**常见状态说明：**
- `device` - 设备已连接并准备就绪
- `offline` - 设备离线
- `unauthorized` - 设备未授权（需要在设备上确认 USB 调试授权）
- `no devices` - 没有检测到设备

## 编译应用

### 清理并编译 Debug 版本

在项目根目录下执行：

```bash
.\gradlew.bat clean assembleDebug
```

**命令说明：**
- `clean` - 清理之前的编译输出
- `assembleDebug` - 编译 Debug 版本的 APK

**编译过程：**
```
Starting a Gradle Daemon (subsequent builds will be faster)
Configuration on demand is an incubating feature.
<=============> 100% CONFIGURING
<=============> 100% EXECUTING

BUILD SUCCESSFUL in 11s
35 actionable tasks: 13 executed, 22 from cache
```

**编译成功后，APK 文件位置：**
```
app\build\outputs\apk\debug\app-debug.apk
```

### 编译 Release 版本（可选）

如果需要编译发布版本：

```bash
.\gradlew.bat clean assembleRelease
```

Release APK 位置：
```
app\build\outputs\apk\release\app-release.apk
```

## 安装应用

### 安装到已连接的设备

使用 ADB 安装编译好的 APK：

```bash
D:\Androidsdk\platform-tools\adb.exe install -r app\build\outputs\apk\debug\app-debug.apk
```

**参数说明：**
- `install` - 安装 APK
- `-r` - 如果应用已存在，则替换安装（保留数据）

**预期输出：**
```
Performing Streamed Install
Success
```

### 安装到指定设备（多设备情况）

如果连接了多个设备，需要指定设备 ID：

```bash
D:\Androidsdk\platform-tools\adb.exe -s emulator-5554 install -r app\build\outputs\apk\debug\app-debug.apk
```

## 启动应用

### 启动主 Activity

```bash
D:\Androidsdk\platform-tools\adb.exe shell am start -n com.example.funlife/.MainActivity
```

**预期输出：**
```
Starting: Intent { cmp=com.example.funlife/.MainActivity }
```

此时应用应该在设备/模拟器上启动并显示主界面。

## 常用命令

### 一键编译、安装并启动

创建一个批处理脚本 `run.bat`，将以下内容保存到项目根目录：

```batch
@echo off
echo ========================================
echo 正在编译应用...
echo ========================================
call gradlew.bat clean assembleDebug

if %ERRORLEVEL% NEQ 0 (
    echo 编译失败！
    pause
    exit /b %ERRORLEVEL%
)

echo.
echo ========================================
echo 正在安装应用...
echo ========================================
D:\Androidsdk\platform-tools\adb.exe install -r app\build\outputs\apk\debug\app-debug.apk

if %ERRORLEVEL% NEQ 0 (
    echo 安装失败！
    pause
    exit /b %ERRORLEVEL%
)

echo.
echo ========================================
echo 正在启动应用...
echo ========================================
D:\Androidsdk\platform-tools\adb.exe shell am start -n com.example.funlife/.MainActivity

echo.
echo ========================================
echo 应用已成功启动！
echo ========================================
pause
```

使用方法：
```bash
.\run.bat
```

### 查看应用日志

实时查看应用日志（Logcat）：

```bash
D:\Androidsdk\platform-tools\adb.exe logcat -s "FunLife"
```

查看所有日志：
```bash
D:\Androidsdk\platform-tools\adb.exe logcat
```

清除日志缓冲区：
```bash
D:\Androidsdk\platform-tools\adb.exe logcat -c
```

### 卸载应用

```bash
D:\Androidsdk\platform-tools\adb.exe uninstall com.example.funlife
```

### 停止应用

```bash
D:\Androidsdk\platform-tools\adb.exe shell am force-stop com.example.funlife
```

### 查看应用信息

```bash
D:\Androidsdk\platform-tools\adb.exe shell dumpsys package com.example.funlife
```

### 截屏

```bash
D:\Androidsdk\platform-tools\adb.exe shell screencap -p /sdcard/screenshot.png
D:\Androidsdk\platform-tools\adb.exe pull /sdcard/screenshot.png
```

### 录屏

开始录屏（最长 180 秒）：
```bash
D:\Androidsdk\platform-tools\adb.exe shell screenrecord /sdcard/demo.mp4
```

停止录屏：按 `Ctrl+C`

下载录屏文件：
```bash
D:\Androidsdk\platform-tools\adb.exe pull /sdcard/demo.mp4
```

## 故障排除

### 问题 1：找不到 ADB 命令

**错误信息：**
```
'adb' 不是内部或外部命令
```

**解决方案：**
使用完整路径或将 Android SDK 的 `platform-tools` 目录添加到系统 PATH 环境变量：
```
D:\Androidsdk\platform-tools
```

### 问题 2：设备未授权

**错误信息：**
```
device unauthorized
```

**解决方案：**
1. 在设备上会弹出 USB 调试授权对话框，点击"允许"
2. 如果没有弹出，尝试重新连接 USB 或重启 ADB：
```bash
D:\Androidsdk\platform-tools\adb.exe kill-server
D:\Androidsdk\platform-tools\adb.exe start-server
```

### 问题 3：编译失败 - Gradle 错误

**错误信息：**
```
Could not resolve all dependencies
```

**解决方案：**
1. 检查网络连接
2. 清理 Gradle 缓存：
```bash
.\gradlew.bat clean --refresh-dependencies
```
3. 删除 `.gradle` 文件夹后重新编译

### 问题 4：模拟器无法启动

**解决方案：**
1. 检查 BIOS 中是否启用了虚拟化技术（Intel VT-x 或 AMD-V）
2. 确保没有其他虚拟化软件冲突（如 Hyper-V）
3. 尝试使用 Android Studio 的 AVD Manager 重新创建模拟器

### 问题 5：安装失败 - INSTALL_FAILED_INSUFFICIENT_STORAGE

**错误信息：**
```
Failure [INSTALL_FAILED_INSUFFICIENT_STORAGE]
```

**解决方案：**
1. 清理设备存储空间
2. 卸载不需要的应用
3. 在模拟器设置中增加存储空间

### 问题 6：应用崩溃

**解决方案：**
1. 查看 Logcat 日志：
```bash
D:\Androidsdk\platform-tools\adb.exe logcat *:E
```
2. 检查应用权限是否正确配置
3. 确保设备 API 级别符合应用要求（minSdk = 26）

## 性能优化建议

### 加速编译

1. **启用 Gradle 守护进程**（默认已启用）
   在 `gradle.properties` 中添加：
   ```properties
   org.gradle.daemon=true
   org.gradle.parallel=true
   org.gradle.caching=true
   ```

2. **增加 Gradle 内存**
   ```properties
   org.gradle.jvmargs=-Xmx4096m -XX:MaxMetaspaceSize=512m
   ```

3. **使用增量编译**
   只编译不清理：
   ```bash
   .\gradlew.bat assembleDebug
   ```

### 快速安装

使用 `installDebug` 任务一步完成编译和安装：
```bash
.\gradlew.bat installDebug
```

## 附录：环境变量配置

为了简化命令，建议将以下路径添加到系统 PATH 环境变量：

1. **Android SDK Platform Tools**
   ```
   D:\Androidsdk\platform-tools
   ```

2. **Android SDK Emulator**
   ```
   D:\Androidsdk\emulator
   ```

3. **Java JDK**
   ```
   D:\Java\jdk-17\bin
   ```

配置后，可以直接使用简化命令：
```bash
adb devices
emulator -list-avds
java -version
```
